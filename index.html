NoiseMap NYC

Predicting Urban Neighbourhood Noise for Apartment Renters
Problem Statement

Urban renters often evaluate neighbourhood noise levels based on short‑term experiences or real‑time information. However, a neighbourhood that seems quiet in February may have an active construction site by May when the tenant moves in. Such changes matter because chronic exposure to noise can harm health and reduce quality of life. Moreover, research shows that levels above 55 decibels can also lower apartment rental values 2% per unit, making noise both a health and financial concern for renters. Despite these risks, existing apps only display static, current noise conditions and lack tools to assess how neighbourhood noise may change in the future, nor do they provide a breakdown of specific noise sources.

Objectives
  • 1Predict neighbourhood noise levels up to 12 months ahead for every 200×200 metre grid block across New York City
  • 2Decompose each prediction by noise source using SHAP(SHapley Additive exPlanations), identifying the contribution of traffic, construction and baseline urban activity
  • 3Present noise level predictions and dominant noise sources per grid block as human-readable labels accessible to non-technical users
  • 4Provide renter-specific tools to assess noise risk within their specific tenancy period
  • 5Deliver all outputs through an interactive web map
Related Works
HowLoud
Provides a location-based Soundscore derived from modelled environmental noise sources.
NYC Noise Complaint Map
Visualises compiled 311 noise complaint data across NYC neighbourhoods as a historical reference.
SONYC (NYU)
Uses a network of acoustic sensors and machine learning to monitor and classify urban noise sources in real time across New York City.
These existing works focus on current or historical noise conditions. None provide forward-looking predictions, source-level attribution via explainable AI or renter-specific decision tools, the gaps this project addresses.
Proposed Approach
Data Processing
1
Data Ingestion
Raw data from five open sources is ingested and read into a consistent format using Pandas. Inputs include construction permit filings, traffic counts, zoning data, noise complaints, and SONYC sensor readings.
2
Preprocessing
Raw data is cleaned and standardised into a consistent format compatible with the spatial processing and modelling pipeline.
3
Spatial-Temporal Snapping
GeoPandas is applied to snap all data sources to a standardised 200×200 metre grid aggregated by month, mapping every complaint, permit, and sensor reading to a unified grid matrix.
4
XGBoost Model Training
An XGBoost regression model is trained on the historical matrix to learn the relationship between urban activity inputs and actual decibel levels. The model then generates 12-month predictions using approved future permits and seasonal traffic projections as forward-looking inputs.
5
SHAP Attribution
SHAP is applied to decompose each prediction into the exact decibel contribution of each noise source including traffic, construction and baseline urban activity and surfaced as human-readable badges.
Data Visualisation

The output is displayed through an interactive map built in Streamlit and PyDeck. Grid Blocks are color coded by predicted noise level and users can click any grid to view SHAP source breakdown alongside renter specific tools including a noise prediction timeline, lease overlap calculator and neighbourhood comparison.

Features
Prediction Map
Colour-coded 200×200m grids with current and monthly time toggle
Noise Source Badges
Construction, Traffic, Baseline labels derived from SHAP attribution per grid

Line chart showing predicted noise trajectory across the 12 months forecast window

Enter move in date and lease length to highlight predicted noisy months
Neighbourhood Comparison
Side-by-side noise profiling of two addresses across all prediction features
Confidence Indicator
High, Medium, Low reliability rating based on sensor and permit data availability
